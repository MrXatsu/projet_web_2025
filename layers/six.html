Couche 6 : Présentation

La couche présentation rend les données présentables. Elle les traduit, compressent et les déchiffrent pour que l’application puisse les utiliser. Comme deux appareils uniques peuvent chiffrer leurs données différemment, il est essentiel que la couche soit capable de traduire sous une syntaxe compréhensible pour le récepteur en lui envoyant le chiffrement de l’expéditeur.

Fonctions principales

  1. Traduction de formats de données

    • Convertit les données d’un format d’application (par ex. JSON, XML, JPEG, MP3…) vers un format standard de transmission, et inversement.
    • Ex : un fichier texte encodé en UTF-8 peut être converti en ASCII selon les besoins du destinataire.

  2. Chiffrement / déchiffrement

    • Gère la sécurité des données en les chiffrant avant envoi et en les déchiffrant à la réception.
    • Ex : TLS/SSL peut être partiellement considéré comme intervenant à cette couche.

  3. Compression / décompression

    • Réduit la taille des données pour accélérer leur transmission.
    • Ex : les images sont compressées (JPEG, PNG), les vidéos aussi (MPEG, MP4), parfois à cette étape.

Exemples de formats ou technologies associés à la couche Présentation :

Type de transformation Exemples
Formats de données JSON, XML, CSV
Encodages texte ASCII, UTF-8, EBCDIC
Compression ZIP, GZIP, MP3, JPEG
Chiffrement TLS, SSL, AES, RSA
Note : ces technologies ne sont pas toujours strictement limitées à la couche 6 dans les modèles pratiques comme TCP/IP. Mais dans OSI, elles y sont classées pour des raisons de logique fonctionnelle.
