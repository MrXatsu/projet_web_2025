Couche 4 : Le transport

La couche Transport récupère les données à partir de la Session et les organisent en segments avant de les envoyer vers la couche 3, Réseau et inversement après que les données arrivent sur l’appareil récepteur. Elle est responsable de contrôler le flux et reporter les erreurs inter-réseau en s’assurant que les données reçues sont complètes.

Fonctions principales

Fiabilité des communications

  • Assure que les données arrivent complètes et dans le bon ordre (si le protocole utilisé le permet)
  • Vérifie la bonne réception des segments grâce à des accusés de réception

Segmentation et réassemblage

  • Découpe les données en segments avant de les transmettre à la couche réseau
  • Le destinataire réassemble les segments pour reconstituer le message complet

Gestion des ports

  • Utilise des numéros de port pour identifier l’application source et destination (ex: port 80 pour HTTP, 443 pour HTTPS)
  • Permet la multiplexion (plusieurs applications peuvent communiquer sur la même machine en même temps)

Contrôle de flux

  • Régule la quantité de données envoyées pour éviter de surcharger le destinataire

Détection et correction d’erreurs

  • Calcule un checksum pour vérifier l'intégrité des segments
  • Peut retransmettre les segments perdus (avec TCP)

Contrôle de congestion

  • Certains protocoles (comme TCP) détectent une congestion du réseau et réduisent le débit pour limiter les pertes

Protocoles

TCP - Transmission Controle Protocol

  • Caractéristiques principales :
    • Fiable: il garantit que les données arrivent correctement et dans le bon ordre.
    • Orienté connexion: une connexion est établie avant l'échange de données.
    • Contrôle d'erreurs: chaque segment contient un checksum pour vérifier son intégrité
    • Contrôle de flux: il ajuste l’envoi de données selon la capacité du récepteur.
    • Contrôle de congestion: réduit le débit si le réseau est saturé
    • Numérotation des segments: chaque segment est numéroté pour permettre le réassemblage
    • Accusés de réception (ACK): chaque segment reçu est confirmé par le destinataire
  • Utilisation typique :
    • Web (HTTP, HTTPS)
    • E-mail (SMTP, IMAP, POP3)
    • Transfert de fichiers (FTP)
    • Connexions distantes (SSH, Telnet)
  • Exemple simplifié
    1. L’ordinateur A envoie un paquet
    2. L’ordinateur B répond : « bien reçu ! »
    3. Si B ne répond pas, A renvoie le paquet
      4. ↪️Transmission fiable mais plus lente, car il y a beaucoup d’échanges de contrôle

UDP - User Datagram Protocol

  • Caractéristiques principales :
    • Non fiable: pas d’accusé de réception, pas de contrôle d’ordre
    • Sans connexion: envoie les données sans établir de lien préalable
    • Pas de surcharge de gestion: donc plus rapide, mais il peut y avoir des pertes
    • Utilise aussi les ports pour identifier les applications
  • Utilisation typique :
    • Streaming audio/vidéo (YouTube, Spotify…)
    • Jeux en ligne
    • VoIP (appels Internet)
    • DNS (résolution de noms)
  • Exemple simplifié :
    1. L’ordinateur A envoie un paquet.
    2. L’ordinateur B le reçoit (ou pas)
      3. ↪️Pas de vérification, pas de retransmission → rapide mais incertain.
